A) IMPORTANT NOTES - make sure you read this! - top

Please note that the RL installation and configuration is not complicated but because it has a lot of customizable settings it might seem complicated, please go through the entire documentation before trying to install it into your own page. Basically what it must be done is to copy some html and js code from the examples we provided and paste it into your own html page and of course add your own content.

When testing local deeplinking is disabled regardless of the RL settings

If you want to use video or audio only, mp4 video and mp3 audio are required. It will work in all browsers on desktop machines or mobile devices, this is extremely important because there is no need to create multiple audio or video formats for browsers that don't support mp4 or mp3, also please note that the mp4 and mp3 video aud audio formats are recommended and used by all major browsers distributors like Google, Microsoft, Apple, Mozilla etc.!

The server is character case sensitive so make sure that the RL settings are identical to those from the provided examples.

Some of the features will not work locally like the share button or playing youtube videos on some browsers like IE and Opera, this is normal behavior because most browsers do not allow this, once uploaded on the server it will function properly.

When something is wrong with a RL action a red info box with the problem description will appear at the window top left corner, please note that this box is there to help you understand the problem, for example if an image is not found the box will appear and inform you about the problem. The box will hide when navigating to another item.

The RL skin is constructed from png images. We have included psd files with slices just in case you want to modify the skin. If you want a different skin the .png files must be modified. We have a great designer in our team so if you need a custom skin please contact us!

C) Constructor parameters - top

Please open minimal-skin-mixed-content.html with a text editor as reference. These parameters represents the possible setting of the RL they are all visiblebelow and described below. The parameters that are highlighted in red means that they can't be changed at run time.

//main settings
  • mainFolderPath:"content" - The main folder path. This folder contains some important files like the skins, audio and video flash players, etc.In this folder you need to add your content (images, videos, etc), please note that your content can be added outside this folder but is a good idea to keep everything centralized.
  • skinPath:"minimal_skin_dark" - The skin path, the name of the folder that contains the skin.
  • facebookAppId:"213684265480896" - this represents the APP id used by facebook to load Facebook albums, for more info about how to get your own facebook APP id and Setup Facebook app please read Setup Facebook app.
  • rightClickContextMenu:"default" - this can be developer, default or disabled. We would appreciate it if you can leave this feature set to developer.
  • buttonsAlignment:"in" - this can be in or out. If set to in the buttons will be positioned near the image. If set to out the buttons will be positioned at the right or left side of the screen.
  • useDeepLinking:"yes" - this can be yes or no. Allow or not deeplinking, if deeplinking is disabled the share button button will share the current page url.
  • useAsModal:"yes" - this can be yes or no. Enable or disable the modal mode (modal means that the lightbox can be closed using the only close button).
  • slideShowAutoPlay:"yes" - this can be yes or no. Enable or disable the RL slideshow autoplay.
  • addKeyboardSupport:"yes" - this can be yes or no. Enable or disable the keyboard left and right arrows to navigate between RL items.
  • showCloseButton:"yes" - this can be yes or no. Hide or show the close button.
  • showShareButton:"yes" - this can be yes or no. Hide or show the share button.
  • showZoomButton:"yes" - this can be yes or no. Hide or show the image zoom in and out button.
  • showSlideShowButton:"yes" - this can be yes or no. Hide or show the slideshow button.
  • showSlideShowAnimation:"yes" - this can be yes or no. Hide or show the slideshow animation.
  • showNextAndPrevButtons:"yes" - this can be yes or no. Hide or show the next and prev buttons.
  • showNextAndPrevButtonsOnMobile:"yes" - this can be yes or no. Hide or show the next and prev buttons on mobile, this is an addition for the showNextAndPrevButtons feature.
  • buttonsHideDelay:3 - this can be yes or no, delay in seconds until the buttons will hide.
  • slideShowDelay:6 - slideshow duration in seconds.
  • defaultItemWidth:800 - the default RL item width.
  • defaultItemHeight:500 - the default RL item height.
  • itemOffsetHeight:50 - a number that represents the px to remove from the top and bottom part of the RL item, think of this as margin top and margin bottom for the RL item.
  • spaceBetweenButtons:1 - a number that represents the space between buttons in px.
  • buttonsOffsetIn:2 - a number that represents the space between the buttons and the main RL item, think of this as margin left for the buttons from the right side of the RL item and margin right for the buttons from the left site of the RL item.
  • buttonsOffsetOut:5 - a number that represents the space between the buttons and window left or right side, think of this as margin right for the buttons from the right side of the RL item and margin left for the buttons from the left site of the RL item.
  • itemBorderSize:4 - a number that represents the RL item border size in px.
  • itemBorderRadius:4 - a number that represents the RL item border radius in px.
  • backgroundOpacity:.9 - a number from 0 to 1 that represents the main background opacity.
  • itemBoxShadow:"none" - this can be none or a box shadow style like "10px 10px 5px #888888", represents the box shadow for the current item.
  • itemBackgroundColor:"#333333" - the RL item background color.
  • itemBorderColor:"#FFFFFF" - the RL item border color.
  • backgroundColor:"#000000" - the RL main background color.
//thumbnails settings
  • showThumbnails:"yes" - this can be yes or no, hides or show the thumbnails.
  • showThumbnailsHideOrShowButton:"yes" - this can be yes or no. Hide or show the show / hide thumbnails button.
  • showThumbnailsByDefault:"yes" - this can be yes or no, shows or not the thumbnails by default when the RL is first showed.
  • showThumbnailsOverlay:"yes" - this can be yes or no, hides or show the thumbnails overlay on hover.
  • showThumbnailsSmallIcon:"yes" - this can be yes or no, hides or show the thumbnails small icon on hover.
  • thumbnailsHoverEffect:"scale" - this can be scale or opacity.
  • thumbnailsImageHeight:80 - a number that represents the thumbnail height in px, please make sure that you are saving your thumbnails images to the height sepecified in this feature.
  • thumbnailsBorderSize:4 - a number that represents the thumbnail border size in px.
  • thumbnailsBorderRadius:0 - a number that represents the thumbnail border radius in px.
  • spaceBetweenThumbnailsAndItem:0 - a number that represents the space between thumbnails and the RL item in px.
  • thumbnailsOffsetBottom:0 - a number that represents an offset is px that pushes the thumbnails up, think of this as margin bottom for the thumbnails.
  • spaceBetweenThumbnails:0 - a number that represents the space between thumbnails in px.
  • thumbnailsOverlayOpacity:.4 - a number from 0 to 1 that represents the thumbnail overlay opacity.
  • thumbnailsOverlayColor:"#000000" - the thumbnails overly color.
  • thumbnailsBorderNormalColor:"#FFFFFF" - the thumbnails normal border color.
  • thumbnailsBorderSelectedColor:"#FFFFFF" - the thumbnails selected color.
//description window settings
  • showDescriptionButton:"yes" - this can be yes or no, hide or show the description button.
  • showDescriptionByDefault:"yes" - this can be yes or no, shows or not the description window by default.
  • descriptionWindowAnimationType:"opacity" - this can be opacity or motion, represents the animation type of the description window when on show / hide.
  • descriptionWindowPosition:"bottom" - this can be top or bottom, represents the description window position inside the RL item.
  • descriptionWindowBackgroundColor:"#FFFFFF" - the background color of the description window.
  • descriptionWindowBackgroundOpacity:.9 - a number from 0 to 1 that represents the description window background opacity.
//video & audio players settings
  • useVideo:"yes" - this can be yes or no. Always set this to no if you don't need mp4 video.
  • useAudio:"yes" - this can be yes or no. Always set this to no if you don't need mp3 audio.
  • videoShowFullScreenButton:"yes" - this can be yes or no, show or hide the video player fullscreen button.
  • addVideoKeyboardSupport:"yes" - this can be yes or no,add or not space keyboard support to pause or play the video (doesn't apply to youtube or vimeo).
  • nextVideoOrAudioAutoPlay:"yes" - this can be yes or no, autoplay for the next video or audio, when changing the video if this is set to yes the video will start playing.
  • videoAutoPlay:"no" - this can be yes or no.
  • videoLoop:"no" - this can be yes or no (doesn't apply to youtube or vimeo).
  • audioAutoPlay:"yes" - this can be yes or no.
  • audioLoop:"no" - this can be yes or no.
  • videoControllerHideDelay:3 - a number that represents the seconds until the control bar is hiding after a period of inactivity.
  • videoControllerHeight:41 -the video control bar height in px.
  • audioControllerHeight:44 -the audio control bar height in px.
  • startSpaceBetweenButtons:10 - a number that represents the space between buttons in pixels.
  • vdSpaceBetweenButtons:4 - a number that represents the space between buttons in pixels.
  • mainScrubberOffestTop:14 - the amount in pixels to push the main scrubber up when the controller is hiding.
  • scrubbersOffsetWidth:10 - a number that represents the total amount in pixels removed from the scrubber bars when they are at the end of the control bar(change this number to understand it better, useful based on the skin type).
  • audioScrubbersOffestTotalWidth:80 - a number that represents the margin right in px for the audio volume bar.
  • timeOffsetLeftWidth:10 - a number that represents the an addition in px to the space between the time indicator left side and the scrubber.
  • timeOffsetRightWidth:10 - a number that represents the an addition in px to the space between the time indicator right side and the neighbor button.
  • volumeScrubberWidth:80 - a number that represents the volume scrubber width in px.
  • volumeScrubberOffsetRightWidth:80 - a number that represents the volume scrubber width in px.
  • videoControllerBackgroundColor:"#FFFFFF" - video control bar background color.
  • videoPosterBackgroundColor:"#0099FF" - video poster background color.
  • audioControllerBackgroundColor:"#FFFFFF" - audio player background color.
  • timeColor:"#000000" - audio and video current and total time color.

D) Setup JSON content playlist - top

Please open with a text editor the minimal-skin-mixed-content.html file as reference.

In this section is explained how to setup a JSON object playlist for the RL to display, please note this object can be created at any time. The JSON playlist object must be defined as a variable on the window and it's variable name has to start with rlobj_, so if you want to create a JSON playlist with the name mixedPlaylist it's final name should be rlobj_mixedPlaylist, this is illustrated in the image below. After the JSON object is created it can be used when RL show method is called (for more info about the show method please read Open / show Revolution Lightbox)

There are ten content types (image, video, audio, youtube, vimeo, flash, google maps, html, ajax and iframe) that RL can display and they are explained in detail below in this section. To add / remove items in the playlist object add or remove items depending of the type that you need. Below is the HTML content playlist with all supported types.

    H) Change lightbox settings at runtime - top

    Please open with a text editor the minimal-skin-mixed-content.html file as reference.

    RL is starting with default settings that are specified in the constructor but it can be reconfigured at run time, for example you can change the background color or the item background color, or have different item background colors for all items. This is done by passing a settings object as a third parameter in the RL show method (for more info about the show method please read Open / show Revolution Lightbox).

    The configuration object must be declared as a variable on the window in a script tag and it can contain all constructor parameters settings except the one that that are highlighted in red, in the image below there is an example with a configuration object that is changing the background color and opacity and sets the buttons alignment to out.

    		//Change settings object
		var props_obj = {
			backgroundColor:"#0099FF",
			backgroundOpacity:1,
			buttonsAlignment:"out"
		}

    After the object is created it can be passed as a parameter to the show method to change the settings, for example when a button is pressed the RL show method is called (examplebelow).

    I) Setup Facebook app (required for loading Facebook albums) - top

    In order for the share button button to function a Facebook app key is required, this key is obtained by setting up a Facebook app as follows:

    • Login into your Facebook account.
    • Open the following link https://developers.facebook.com/apps
    • From the top bar menu select My Apps - > Add New App then select Website and set the app name.
    • After this the app can be found by oppening the My Apps from the menu. In the Settings section set the website url where the grid is used, example in the image below, also in this screen the App ID and App Secret can be seen.
    • Last step is to select from the left menu bar Status & Review, in this page there is a button that is referring to "Do you want to make this app and all its live features available to the general public?" that must be set to yes, examplebelow.

    One important thing to note is that the Revolution Lightbox domain name must be identical with the url passed to the Facebook app, for example if the Revolution Lightbox domain name is http://www.some-domain.com and the Facebook app is configured with the url/domain name http://some-domain.com it will not work, both url / domain names must be exactly the same.

    J) API specifications - top

    Inside the start file there is HTML file called API-example.html, open it with a text editor as reference. Below is a table with all API events.

    Event listeners:

    Event Prefix Description
    showStart FWDRL.SHOW_START Dispatched when RL show animation starts.
    showComplete FWDRL.SHOW_COMPLETE Dispatched when RL show animation is complete.
    hideStart FWDRL.HIDE_START Dispatched when RL hide animation starts.
    hideComplete FWDRL.HIDE_COMPLETE Dispatched when RL is hide animation is complete.
    update FWDRL.UPDATE Dispatched when RL item is changing. This event also has a parameter curId that represents the current item id.
